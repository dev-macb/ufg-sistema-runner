# 🔹 CLI assinador (Go)

## US-01: Executar comando de criação de assinatura (CLI → subprocess)
Como usuário do Runner
Quero executar o comando assinador assinar via terminal
Para que o sistema invoque o assinador.jar via subprocess (java -jar) e gere uma assinatura simulada

## US-02: Executar comando de validação de assinatura (CLI → subprocess)
Como usuário do Runner
Quero executar o comando assinador validar via terminal
Para que o sistema invoque o assinador.jar via subprocess e retorne o resultado da validação

## US-03: Executar comando de criação de assinatura (CLI → HTTP)
Como usuário do Runner
Quero executar o comando assinador assinar configurado para modo HTTP
Para que o CLI envie uma requisição HTTP ao assinador.jar em execução e receba a assinatura simulada

## US-04: Executar comando de validação de assinatura (CLI → HTTP)
Como usuário do Runner
Quero executar o comando assinador validar configurado para modo HTTP
Para que o CLI envie uma requisição HTTP ao assinador.jar e receba o resultado da validação

## US-05: Validar flags obrigatórias no CLI
Como usuário do Runner
Quero que o CLI valide a presença e o formato dos flags obrigatórios (--pacote, --politica, etc.)
Para que erros de entrada sejam detectados antes da execução do Java

## US-06: Selecionar modo de execução do assinador
Como usuário do Runner
Quero escolher entre execução via subprocess ou HTTP
Para que eu possa otimizar entre simplicidade (CLI) e performance (HTTP)

## US-07: Exibir saída padronizada no terminal
Como usuário do Runner
Quero ver o resultado da operação formatado no stdout
Para que eu consiga interpretar facilmente o retorno da assinatura ou validação

## US-08: Exibir erros com códigos de saída padronizados
Como usuário do Runner
Quero receber erros no stderr com exit codes definidos
Para que eu consiga automatizar o uso via scripts

Para que ele valide os dados e retorne uma assinatura simulada

## US-10: Executar operação de validação de assinatura (modo CLI)
Como sistema Runner
Quero invocar o assinador.jar para validar uma assinatura
Para que ele retorne o resultado da validação simulada

## US-11: Disponibilizar API HTTP para operações de assinatura
Como sistema Runner
Quero que o assinador.jar execute como servidor HTTP
Para que múltiplas requisições sejam processadas sem overhead de inicialização

## US-12: Validar parâmetros conforme FHIR
Como sistema Runner
Quero que o assinador.jar valide todos os parâmetros conforme a especificação FHIR
Para que apenas requisições válidas sejam processadas

Como sistema Runner

Como sistema Runner

Para que fluxos de integração possam ser testados

## US-15: Retornar resultado de validação simulada
