# Plano de Execução Detalhado

## Metodologia
- **Sprints**: 4 sprints de 1 semana
- **Abordagem**: Iterativa e incremental
- **Priorização**: Entregar valor funcional o mais cedo possível, removendo riscos técnicos

---

**Objetivo**: Estabelecer infraestrutura de desenvolvimento, CI/CD e estrutura base dos CLIs.

### Tarefas

| ID | Componente | Tarefa | Critérios de Aceitação | Estimativa |
|----|-----------|--------|------------------------|------------|
| **1.1** | **assinador (Go)** | Estrutura inicial do projeto | - Módulo Go inicializado<br>- Comando `assinador version` funcionando<br>- Build cross-compile funcional | 4h |
| **1.2** | **simulador (Go)** | Estrutura inicial do projeto | - Módulo Go inicializado<br>- Comando `simulador status` funcionando (mock)<br>- Build cross-compile funcional | 4h |
| **1.3** | **shared (Go)** | Estrutura de pacotes compartilhados | - Módulo `shared` criado<br>- Pacotes `jdk`, `output`, `executor`, `processo` definidos<br>- Importação funcional nos CLIs | 6h |
| **1.4** | **assinador-jar (Java)** | Estrutura Maven | - `pom.xml` configurado<br>- Dependências do FHIR (HAPI) adicionadas<br>- Estrutura de pacotes definida | 4h |

## Sprint 2: Assinatura Simulada - Modo Local (Semana 2)

**Objetivo**: Fluxo completo de assinatura e validação via invocação local do `assinador.jar`.

### Tarefas

| ID | Componente | Tarefa | Critérios de Aceitação | Estimativa |
|----|-----------|--------|------------------------|------------|
| **2.1** | **assinador-jar** | Modelos de parâmetros | - Classes `CriarAssinaturaParams` e `ValidarAssinaturaParams`<br>- Validação de tipos e formatos | 6h |
| **2.2** | **assinador-jar** | Validação de parâmetros (criar) | - Implementar validação conforme FHIR<br>- Testes unitários para todos os parâmetros<br>- Mensagens de erro em `OperationOutcome` | 8h |
| **2.3** | **assinador-jar** | Simulação de criação | - `FakeSignatureService` implementada<br>- Retorna JWS JSON Serialization pré-construída<br>- Testes unitários | 4h |
| **2.4** | **assinador-jar** | Validação de parâmetros (validar) | - Validar JWS, trust store, timestamp, política<br>- Validar `revocationPolicy`, `ocspUnknownHandling`<br>- Validar timeouts e thresholds | 8h |
| **2.5** | **assinador-jar** | Simulação de validação | - Retorna `OperationOutcome` com sucesso<br>- Testes unitários | 4h |
| **2.6** | **shared** | JDK Provisioning | - Detecção de JDK 21 no sistema<br>- Download automático de JDK (Adoptium)<br>- Armazenamento em `~/.hubsaude/jdk/`<br>- Testes de integração | 6h |
| **2.7** | **shared** | Executor de subprocessos | - Função `ExecutarAssinadorLocal(params)`<br>- Captura stdout/stderr<br>- Tratamento de erros e exit codes | 6h |
| **2.8** | **assinador** | Parsing de comandos | - Comandos `assinar` e `validar`<br>- Flags: `--pacote`, `--politica`, `--jws`, etc.<br>- Help integrado<br>- Validação de flags obrigatórios | 6h |
| **2.9** | **assinador** | Invocação local | - Chamar `shared.ExecutarAssinadorLocal`<br>- Repassar saída para stdout/stderr<br>- Exit codes apropriados | 4h |
| **2.10** | **assinador** | Formatação de resultados | - Saída legível no terminal<br>- Erros estruturados e informativos<br>- Testes de integração CLI → JAR | 4h |
| **2.11** | **Documentação** | Manual de usuário (assinador) | - Comandos, flags e exemplos<br>- Modo local vs servidor<br>- Tratamento de erros | 4h |

### Entregas ao final da Sprint 2
- ✅ `assinador.jar` validando parâmetros e simulando respostas
- ✅ CLI `assinador` invocando localmente o JAR
- ✅ Provisionamento automático de JDK
- ✅ Fluxo ponta-a-ponta funcional: usuário → CLI → JAR → resultado

## Sprint 3: Modo Servidor e PKCS#11 (Semana 3)

**Objetivo**: Servidor HTTP no `assinador.jar` e integração com dispositivos criptográficos.

### Tarefas

| ID | Componente | Tarefa | Critérios de Aceitação | Estimativa |
|----|-----------|--------|------------------------|------------|
| **3.1** | **assinador-jar** | Servidor HTTP | - Endpoints `POST /sign` e `POST /validate`<br>- Reutilizar lógica de validação/simulação<br>- Respostas JSON consistentes<br>- Testes de integração HTTP | 8h |
| **3.2** | **assinador-jar** | Gerenciamento de ciclo de vida | - Inicialização com porta configurável<br>- Graceful shutdown via `/shutdown` endpoint<br>- Timeout de inatividade configurável | 6h |
| **3.3** | **assinador-jar** | PKCS#11 Provider | - Configuração do SunPKCS11<br>- Detecção de dispositivos criptográficos<br>- Simulação com SoftHSM2 | 8h |
| **3.4** | **shared** | Gerenciamento de processos | - Salvar PID e porta em `~/.hubsaude/processos/`<br>- Verificar se processo está ativo<br>- Encerrar processo por PID | 6h |
| **3.5** | **shared** | Cliente HTTP | - Função `ExecutarAssinadorHTTP(params)`<br>- Timeout configurável<br>- Tratamento de erros de rede | 6h |
| **3.6** | **assinador** | Iniciar servidor | - Comando `assinador servidor iniciar`<br>- Flag `--port` (padrão 8080)<br>- Feedback de inicialização | 4h |
| **3.7** | **assinador** | Parar servidor | - Comando `assinador servidor parar`<br>- Encerra processo pelo PID registrado | 2h |
| **3.8** | **assinador** | Modo HTTP automático | - Detectar servidor em execução na porta padrão<br>- Usar HTTP quando disponível, fallback local<br>- Flag `--local` para forçar modo local | 6h |
| **3.9** | **assinador** | Timeout automático | - Flag `--timeout` minutos sem interação<br>- Encerrar servidor após inatividade | 4h |
| **3.10** | **Documentação** | Setup PKCS#11 | - Guia de configuração de dispositivo criptográfico<br>- Exemplos com SoftHSM2 | 4h |

### Entregas ao final da Sprint 3
- ✅ `assinador.jar` funcionando como servidor HTTP
- ✅ CLI detectando e usando servidor em execução
- ✅ Gerenciamento de processos (iniciar/parar/timeout)
- ✅ Suporte a PKCS#11 com testes em SoftHSM2

## Riscos e Mitigações

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Dificuldade com PKCS#11 e SoftHSM2 | Média | Alto | Sprint 3 priorizar setup antecipado; usar container Docker para ambiente controlado |
| Integração CLI → JAR com parâmetros complexos | Baixa | Médio | Testes de integração desde Sprint 2; usar structs compartilhadas para serialização |
| Cross-compilation de bibliotecas CGO | Baixa | Médio | Evitar CGO; usar bibliotecas puras Go; testar em CI para todas as plataformas |
| GitHub API rate limits no download do JDK | Baixa | Baixo | Usar mirror (Adoptium) com fallback; cache local |
| Compatibilidade Windows com processos em background | Média | Médio | Testar com `syscall` específico; usar `exec.Command` com `SysProcAttr` para detached |
